Security Architecture

6 layers of protection.

🔐
AES-256 Encryption
All data at rest is encrypted using AES-256, the same standard used by governments and major financial institutions worldwide.
At Rest & In Transit
🌐
TLS 1.3 Transport
Every byte of data between your device and our servers travels over TLS 1.3 — the latest and most secure transport protocol.
Zero Plaintext Transfer
🎫
JWT Auth + MFA
Short-lived JWT tokens with refresh token rotation. Multi-factor authentication available for all accounts.
Token-Based Auth
🛡️
OWASP Top 10 Hardened
Our codebase is regularly audited against the OWASP Top 10 security risks. SQL injection, XSS, CSRF — all mitigated.
Quarterly Audits
👁️
Zero-Trust Access Control
Internal employee access to production data follows a strict zero-trust model — role-based, time-limited, and fully audited.
Employee Access Logging
🔎
Continuous Monitoring
24/7 anomaly detection, intrusion detection, and automated threat response. Our security team is always watching.
24/7 SOC Monitoring
Technical Specifications

Under the hood.

🔑
Password Hashing
bcrypt with cost factor 12. Passwords are never stored in plain text. Even our engineers cannot read your password.
🗄️
Database Security
PostgreSQL with row-level security. Each user's data is logically isolated. Field-level encryption for sensitive fields.
☁️
Cloud Infrastructure
AWS Mumbai region (ap-south-1). All data stored within India. VPC isolation, security groups, and WAF enabled.
📋
Audit Logs
Immutable audit logs for all data access and modification events. Retained for 12 months. Available to enterprise users.
🔄
Backup & Recovery
Automated daily backups with point-in-time recovery. Backups are encrypted and stored in a separate AWS region.
🚨
Rate Limiting & DDoS
API rate limiting with exponential backoff. AWS Shield Standard for DDoS protection on all endpoints.

Responsible Disclosure

Found a security vulnerability?

We take security seriously and appreciate the responsible disclosure of vulnerabilities. If you've found a security issue in SmartSpend, please report it — we'll respond within 48 hours and publicly credit you (with your permission).

✓ Email details to security@smartspend.in
✓ Include steps to reproduce, severity assessment, and any proof-of-concept
✓ Please allow us 90 days to patch before public disclosure
✓ We will never pursue legal action against good-faith researchers
